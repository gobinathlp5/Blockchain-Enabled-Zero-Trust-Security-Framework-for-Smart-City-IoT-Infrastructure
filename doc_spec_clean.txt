# PARAGRAPHS
1. Project Overview
This project implements a Blockchain and Zero Trust Architecture (ZTA) security framework on two ESP32 microcontrollers to protect simulated smart city IoT infrastructure from cyber threats. ESP32 #1 acts as a multi-role IoT sensor simulator generating signed data packets. ESP32 #2 acts as a Zero Trust Gateway that verifies every incoming message and logs approved data on an on-device blockchain ledger.
1.1 Core Objectives
Demonstrate Zero Trust authentication on resource-constrained IoT hardware
Implement HMAC-SHA256 cryptographic signing on every data transmission
Build a SHA-256 hash-chain blockchain ledger entirely on ESP32
Simulate real smart city attack scenarios: rogue devices, tampered data, DDoS flooding
Show real-time security decisions on Serial Monitor as an admin dashboard
1.2 Why Two ESP32s With No Sensors
Removing the physical DHT11 sensor eliminates hardware dependency and makes the demo fully software-controlled. ESP32 #1 can now simulate any smart city device type — environmental sensor, traffic monitor, smart meter — by simply changing the data payload. This approach lets the demonstration focus entirely on the security layer, which is the core contribution of the case scenario.

2.3 Data Flow Summary
ESP32 #1 cycles through four roles every 30 seconds: Environmental Node → Traffic Node → Smart Meter Node → Rogue Device
Each legitimate message is signed with HMAC-SHA256 before transmission
ESP32 #2 runs three Zero Trust checks on every received message
Passing messages are hashed and appended as a new blockchain block
Failing messages trigger a quarantine alert logged to Serial Monitor
3. Simulated Device Types
ESP32 #1 simulates four distinct device identities. Each has a unique Device ID, data payload, and behavior pattern.
